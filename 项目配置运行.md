# 基金研究子系统 — 配置运行手册

> 适用平台：Windows 10/11 + PowerShell 5.1；Linux/macOS 仅需把 `powershell` 命令替换为 `bash`

## 1. 项目简介

智能投顾基金研究平台——面向 C 端用户的基金研究 / 智能选基决策子系统。开源仓库地址：<https://github.com/galaxylove456789-create/fund-research-system.git>。本仓库包含前端、后端、KingbaseES Docker 数据库、HarmonyOS 端和部署文档：

| 子项目 | 类型 | 主要技术 | 端口 |
| --- | --- | --- | --- |
| `fund_research_backend` | 后端 REST API | Spring Boot 3.2.5 / Java 17 / MyBatis-Plus 3.5.10 / KingbaseES V9 | **8080** |
| `fund_research_front` | PC Web 前端 | Vue 3 + Vite + Pinia + Element Plus + ECharts，默认调用后端真实接口 | dev **5173** / 生产 **80** |
| `docker/kingbase` | KingbaseES 数据库容器 | KingbaseES V9 + 初始化 SQL | **54321** |
| `harmony_fund_research` | HarmonyOS 端 | ArkTS / DevEco Studio | — |
| `docker-compose.yml` | 一键编排 | Docker Compose v2 | 80 + 8080 + 54321 |

## 6. 数据库初始化（KingbaseES V9）

### 6.1 推荐方式：Docker 自动初始化 KingbaseES

当前仓库已经自带 KingbaseES Docker 封装和完整初始化 SQL，不再依赖课外单独提供的建表脚本。首次执行 `docker compose up -d --build` 时，`FundResearch-kingbase` 容器会自动创建 `fund_research` 数据库并执行：

```text
docker/kingbase/initdb/002_previous_full_database.sql
docker/kingbase/initdb/003_auto_sequences.sql
```

其中 `002_previous_full_database.sql` 用于恢复项目已有业务数据，`003_auto_sequences.sql` 用于补齐自增序列，保证注册、收藏、组合、对比、AI 记录等后续写入正常。

数据库默认连接信息：

```text
host: localhost
port: 54321
database: fund_research
username: system
password: 1234
```

### 6.2 手动安装 KingbaseES（可选）

如不使用 Docker，也可以手动安装 KingbaseES V9。安装后创建 `fund_research` 数据库，并按顺序执行 `docker/kingbase/initdb/` 下的 SQL 文件。远程访问时需要在 `kingbase.conf` 中配置 `listen_addresses = '*'`，并确认防火墙开放 `54321` 端口。

### 6.3 手动运行 Python 数据脚本（可选）

仓库不提交 `.venv`。只有在需要重新抓取或补充数据时，才需要在 `fund_research_backend` 下创建虚拟环境：

```powershell
cd <ROOT>\fund_research_backend
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install --upgrade pip
.\.venv\Scripts\python.exe -m pip install psycopg2-binary pandas akshare requests python-dotenv
```

> Docker 初始化已经包含完整演示数据；Python 脚本属于后续数据维护工具，不是首次运行项目的必需步骤。

## 7. 启动方式（按场景选一）

### 7.1 方案 A — Docker Compose 一键编排 ⭐

> 推荐方式。Docker Compose 会同时启动 KingbaseES、后端和前端。Dify 是外部服务，如需 AI 推荐解释，请单独启动 Dify 并在 `.env` 中配置 `DIFY_BASE_URL` 与 `DIFY_API_KEY`。

```powershell
cd <ROOT>
# 1) 编辑 .env
Copy-Item .env.example .env -Force
notepad .env

# 2) 构建 + 启动
docker compose up -d --build

# 3) 查看日志
docker compose logs -f backend
docker compose logs -f frontend

# 4) 停止
docker compose down
```

容器映射：

| 服务 | 容器名 | 暴露端口 | 内部地址 |
| --- | --- | --- | --- |
| kingbase | `FundResearch-kingbase` | `54321 → 54321` | `jdbc:kingbase8://kingbase:54321/fund_research` |
| backend | `FundResearch-backend` | `8080 → 8080` | 连接 `kingbase:54321` |
| frontend | `FundResearch-frontend` | `80 → 80` | 反代 `/api/` → `http://backend:8080` |

访问：`http://localhost/`（前端） / `http://localhost:8080/doc.html`（Knife4j 文档）。

### 7.2 方案 B — 本地分别启动（开发推荐）

需要打开 **3 个 PowerShell 窗口**。

#### 7.2.1 终端 1：后端 Spring Boot

```powershell
cd <ROOT>\fund_research_backend
$env:SPRING_PROFILES_ACTIVE = "local"
$env:FUND_DB_PASSWORD     = "1234"
# $env:DIFY_API_KEY         = "app-xxx"   # 有 Dify 时再设

mvn -DskipTests spring-boot:run
# 或：mvn -DskipTests package  → target\fund-research-backend.jar → java -jar ...
```

成功标志：日志看到 `Tomcat started on port 8080`，浏览器访问 `http://localhost:8080/api/v1/health` 返回 `{"code":0,...}`。

#### 7.2.2 终端 2：前端 Vite 开发服务器

```powershell
cd <ROOT>\fund_research_front
npm install                 # 首次安装，约 1～3 分钟
npm run dev
```

打开提示的 `http://localhost:5173`。`/api` 已被 Vite 代理转发（默认目标 `http://localhost:8084`，如后端跑在 8080，请另开窗口设置代理目标）：

```powershell
$env:VITE_DEV_API_TARGET = "http://localhost:8080"
npm run dev
```

#### 7.2.3 前端接口说明

当前前端默认通过 `src/api/request.js` 调用后端真实接口，`src/mock/` 目录仅作为历史演示数据和离线调试参考，不作为默认数据源。开发时如果后端运行在 8080，需要设置：

```powershell
$env:VITE_DEV_API_TARGET = "http://localhost:8080"
npm run dev
```

### 7.3 方案 C — 后端跑在 8084（Vite 默认代理目标）

```powershell
cd <ROOT>\fund_research_backend
$env:SERVER_PORT          = "8084"
mvn -DskipTests spring-boot:run
```

然后前端**无需设置 `VITE_DEV_API_TARGET`**，默认指向 8084。

### 7.4 方案 D — 仅查看前端页面（不推荐作为验收方式）

如果只想查看页面布局，可以只启动前端开发服务器；但注册、登录、基金查询、智能推荐、对比、自选和后台导入等核心功能都依赖后端与 KingbaseES。验收和答辩建议使用方案 A。

## 11. 常见问题（FAQ）

| 现象 | 处理 |
| --- | --- |
| `Connection refused :54321` | 启动 KingbaseES；确认本地 `ksql -Usystem -p1234 -h127.0.0.1 -p54321 fund_research` 可登录 |
| 中文乱码 | PowerShell 输入 `chcp 65001`；脚本加 `$env:PYTHONIOENCODING="utf-8"` |
| `psycopg2.OperationalError` | Windows 安装 psycopg2 时改用 `pip install psycopg2-binary` |
| npm install 慢 / 失败 | 配置 `npm config set registry https://registry.npmmirror.com`，并清缓存 `npm cache clean --force` |
| `mvn package` 失败 | 确认 JDK 版本：`mvn -v` 中 `Java version` 必须是 17；删 `target/` 重试 |
| Knife4j 打不开 | 后端是否启动；`knife4j.enable=true` 是否生效（已在 `application.yml`） |
| `DIFY_API_KEY` 留空 | `/api/v1/ai/*` 返回 `business error: DIFY_DISABLED` 是预期行为，不影响其它模块 |
| Kingbase 容器首次启动较慢 | 首次执行完整 SQL 需要等待，使用 `docker compose logs -f kingbase` 查看初始化进度 |
| 前端 5173 起不来 | `node_modules/` 损坏 → 删掉后 `npm install` 重新生成 |
| 后端 8080 被占用 | 修改 `application.yml` 的 `server.port`，或在 IDE 启动配置里加 `--server.port=8081` |

## 12. 安全注意事项（项目硬性约定）

1. **Dify Key / 数据库密码 / JWT Secret** —— 严禁硬编码到代码或前端。
2. **不要提交** `application-prod.yml`、`application-local-real.yml`、`*.env`（已在根、子工程 `.gitignore` 中屏蔽）。
3. 前端不会直接连数据库，所有数据通过 `/api/v1/*` 访问。
4. Controller → Service → Mapper 三层结构，禁止 Controller 直接调用 Mapper。
5. Entity 不直接对外，DTO 入参、VO 出参。
6. 所有列表接口必须分页，大整数 ID 用 `@JsonSerialize(using = ToStringSerializer.class)` 序列化为字符串。
